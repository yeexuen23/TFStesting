CHAPTER 23 cursor definitions

23-1 appearance

23-1.1 what they are

In the old days, people used to count graticule lines or even mark the CRT with a grease pencil to aid in interpreting the waveform. While some users still do, most people have discovered cursors. Cursors are dashed lines in the display, which can be moved by the entry knob. The position of the 4 available cursors, relative to a source, is listed on the screen. This position is in the scale and units of its associated channel, usually a time or voltage.

23-1.2 units

vertical cursors can have the units of Seconds, Degrees.

horozontal cursors can have the units of Volts, IRE, percent.

in XY mode, vertical cursors can have Volt units.

with the FFT, horozontal cursors are in dBV, and vertical cursors are in Hz.

vertical cursors on digital data can have binary or hex readout as well as time units.

23-2 modes

23-2.1 manual

In manual mode the UI can turn on a cursor, choose its source, and move it around the screen with the entry knob. It is shown in the main timebase window, and if it would be visible also in the delayed window it is shown there.

The user needn't be in the cursor menu to adjust manual cursors. If the menu doesn't use the entry knob itself, and there is not a measurement in the measurement list, and there is an active cursor, then the cursors are displayed and the entry knob adjusts the active one. In that case, the active cursor is indicated in the menu.

23-2.2 showMeas

If the user has chosen "show meas" in the measurement system, a set of cursors is positioned by the last measurement chosen. When the measurement is updated, these cursors are updated also, so they appear live. The measurement line gives measurement output, so the cursors are visible as lines, not as values. Measurement cursors are not amenable to the entry knob. If the user goes to the cursor menu, cursor values are displayed instead of the measurements, and the mode is back to manual.

23-2.3 tracking (not JEF)

These are live cursors, but without a corresponding measurement result. Implementing them would require some innovation in the UI. We will do something like a tracking cursor for the "find peaks" function on an FFT.

23-2.4 serial

For I2C and SPI protocols, digital data can be read out in hex. May not include analog data. We will support whatever serial modes we are doing in the trigger, and then some. There will be a separate setup menu for the cursor, since its features may diverge from the trigger. Someday there could be some hardware support for this decoding, possibly in ZEUM.

23-2.5 source considerations

Cursors can go on the screen as dashed lines, independent of data.

Vertical cursors can give time readouts without any channels being on, since horizontal positioning is the same for all channels.

Manual horozontal cursors need the source to be declared and viewed, so they can access its scaling.

Any cursors displayed by the showMeas option of measurements must have the source viewed and the measurement successful. It is up to the measurement to cloak them if it failed.

So sources can be 1 2 3 4 D0 - DF 1+2 1-2 1*2 FFT d/dt integral, and I may need a bitmap for d/dt and integral. I see no reason to call it F2. For filter output, we may need to label these sources with something other than d/dt or 1-2.

This is a superset of the sources for functions.

23-2.6 improving the FFT cursors

When the cursor source is a live FFT, the cursor menu gets some extra controls: find peaks, and move F1 to center. I think this is confusing, because find peaks is really a measurement with tracking cursors, and move F1 to center is really a horozontal pan. Maybe it can be improved in Muskrat. These controls should probably all be in the FFT menu, or in the measurement menu??

You hit "find peaks" and the horizontal cursors go to the two highest peaks, then the vertical cursors read the level at those frequencies. In JediPlus, this will be a tracking function, updated at a measurement rate. We should never find DC as a peak.

The units of FFT are dBV, or dB below 1.0 V. Should we use dBm instead, especially in a 50-ohm system? Or dB below the signal amplitude?

In Moab, the "move f1 to center" button is in the cursor menu, but the "move 0Hz to left" is in the FFT menu. huh??

One's ability to horizontally pan and zoom the FFT window is limited.

There is a small set of allowable Freq Spans, which you can choose with the knob or the softkey in the FFT menu. Yet when you hit the softkey at the highest value, it will not wrap around to the smallest value.

The frequencies listed are artifacts of the FFT, even though the whole FFT output is not displayed so the user is in a horizontal vernier anyway, and these frequencies are probably not what the user thinks of.

I mean, like, I put in a nice 10 MHz sine, dial in a nice 500 nS/division, and wonder what the peak is, at 97.66 MHz span? Find peaks (in the cursor menu) says it is 9.961 MHz, outrageous precision for this transform. Or do users expect such difficulties with Fourier Transforms?

Clearly, the UI for FFT control should be improved. I would take it out of the cursor menu, except for manual cursor control. I think find peaks is a live measurement, and horizontal scaling should be in the FFT menu.

23-3 software support to screenMgr

23-3.1 manual scaling

The UI can move manual cursors around the screen, but the screenMgr must interpret their values. Similarly, the measurements can say what values to set the cursors to, but the screenMgr must figure out where to put them on the screen. Tracking cursors are a special case of a measurement. In all these cases, we need some scaling and interpretation functions to help the labeling and placement of these dotted lines. This information may be readily available in the signal structure from the acqMgr, but if not the signal analysis library should include these ancillary functions: 

 mStat getVScaling (sourceType c, *org, *inc, *units);
 mStat getHScaling (sourceType c, *org, *inc, *units);

Presumably, if screenMgr knows where it is putting things on screeen, then having this information about the data will allow it to label where its cursors are, in data space.

23-3.2 updating the serial cursor readout

Interpreting the time cursors serially is run as a measurement, strobed by the scheduler, and output via a call to the screenMgr in something like updatSerialCursors().
