1.0 DESCRIPTION

The AutoProbe Interface is a probe interface for both active and passive probes that has been designed to provide an intelligent, automated connection to Hewlett Packard/Agilent Technologies products. The AutoProbe Interface provides power and control of AutoProbe compatible probes while setting up the appropriate operator interface on the product to which the probe is connected. The AutoProbe Interface also supports existing passive probes that employ a probe identification resistor, Probe ID. This External Reference Specification includes mechanical and electrical specifications making it possible for anyone (even outside Hewlett Packard/Agilent Techologies) to design an AutoProbe compatible probe. The AutoProbe Interface includes a BNC connector for coupling the main signal at frequencies up to 4 GHz. A nine-pin "pogo'pin" connector provides +/- 12 volts, tracking programmable voltages of +/-3 to +/- 6 volts, a voltage programming resistor , Probe ID, a 2-wire bus and offset current. The number of AutoProbe Interfaces on a given product would generally range between 1 and 4, but is not limited.

An AutoProbe Lite Interface is an interface that does not meet the full requirements of the AutoProbe Interface specification. A common implementation of a Lite interface is where there may be 4 interfaces and the total power to these interfaces will power only 1 or 2 full blown AutoProbe probes or maybe 2 of the 4 interfaces don't have offset capability.

These 2 voltages are supplied by the oscilloscope or other product. To ensure low noise and low output ripple voltages, ripple rejectors or series regulators would normally be used to supply this power.

2.5 Probe power consists of four voltages: +/- 12 volts and +/- 3 to 6 volts. Very clean (i.e. low noise) +/- 12 volt buses are supplied by ripple rejectors on the acquisition board. These ripple rejectors also provide current limiting of approximately 1.6 amps which has duty cycle foldback current limiting. There are two tracking series regulators for each of the four probe interfaces. They provide the +/- 3 to 6 volts. This voltage is hardware programmable via a resistor in each probe pod. Current limiting is provided by these two series regulators.

Each probe I/F has offset capability. A programmable current source provides up to +/- 1 mamp of offset current. A resistor in the pod makes the current-to-voltage conversion and references the offset voltage to the probe common. It is recommended that a 0.47 uF capacitor be placed in parallel with the resistor to provide filtering. The TL074 opamps are used as comparators to calibrate for 0 output current (when probes are not connected).

Each probe I/F has a clock and data line for two way communication between the probe and the oscilloscope. This interface is I^2C compatible but can also provide various, other, simpler protocols. Alternatively, both lines can be used as control lines, status lines or interrupt lines.

A 16-channel, 16-bit DAC, 1821-1690 (MULTI-DAC), is used to provide offset and an ohmmeter function (for measuring the probe ID resistors) for each of the four probes. Four DAC channels provide offset via a two-pole filter and a voltage-to-current converter implemented using 1/2 of an LT1112 opamp. One DAC channel is used to generate a quad 2 to 100 uamp current source for exciting the probe ID resistors. Another DAC channel is used to determine the resultant voltage across the probe ID resistors (by use of LM339A comparators). Probe ID resistors should be measured using two values of current (e.g. 25 uamps and 100 uamps) to remove measurement errors caused by comparator offset voltage, transistor VBE differences, resistor tolerances and probe ground offset voltage errors. One DAC channel must be used as a reference channel for the other 15 channels. Nine DAC channels are unused.

An FPGA is used to convert the serial data from the acquisition board to control, data and clock lines to be used by the PROBE CONTROL AND POWER BOARD and the four probe interfaces. It also sends data, clock and interrupt signals to the acquisition board. The FPGA provides the I^2C (or other) serial interface to the probes, and reads data from the probes. The FPGA generates an interrupt (that is sent to the oscilloscope) when a probe has been connected to or disconnected from the oscilloscope.

The Probe ID resistor (located in active and passive probes) shall identify certain characteristics of the probe. Look-up tables in the oscilloscope will allow the oscilloscope to display a number of standard "screen setups" for the more common active and passive probes. e.g. a 10:1 passive probe requiring a 1 megohm input impedance would use a 9.09k ohm Probe ID resistor and a 1:1 active probe requiring a 50 ohm input impedance and offset capability would use a 3.48k ohm Probe ID resistor. The look-up tables will also contain setup information for existing (non-AutoProbe) active probes that do not fit into the "standard" category. The user may be allowed to change the "oscilloscope user unique" default settings in the Probe ID Resistor Table. e.g. if a user has a probe with a Probe ID resistor value that doesn't agree with the table, he/she can modify the default for that value Probe ID resistor so that the oscilloscope will set up to the new default values whenever that value is detected. A digital Probe ID can be read using the I^2C interface. Future probes that don't fit the "standard" category can use the digital Probe ID or be assigned a unique Probe ID resistor value (which would require re-programming the oscilloscope to support that probe). When the oscilloscope power is turned on, it will be necessary to poll the probes to know how to set up the oscilloscope because probes may have been changed when power was off.

It may be advisable for the oscilloscope to display messages when a probe is changed. One might be, "Please check the compensation range of the probe to see if it is compatible with the oscilloscope input capacitance." (this applies to passive probes) Another message might be, "Please compensate the probe." (again, it applies to passive probes) A third might be the reminder, "Please calibrate the probe."

Note 4. The DC resistance between the +/- VPRG regulators and the AutoProbe Interface pads shall not exceed 0.4 ohms.

3.2 HOT SWAP ISSUES

AutoProbe probes are intended to be installed and removed on oscilloscopes and other instruments when these instruments are turned on. That means that the AutoProbe slot is live when the probe is attached. This is often referred to as a "hot swap" installation and removal. During hot swap the power supply bypass capacitors in the probe can draw excessive transient currents from the AutoProbe interface power buses on the oscilloscope or other instrument. These transient currents can cause glitches on the voltage buses of the AutoProbe interface which could result in a fault in the oscilloscope or other instrument, and could result in permanent damage to capacitors or other components. The following table identifies capacitance limits on each of the AutoProbe voltages where there is essentially no resistance in series with the capacitors.
Maximum Capacitance
Voltage Capacitance (UF)
+12V 6
-12V 6
+Vp 27
-Vp 27

If capacitance exceeds the values in the Maximum Capacitance table, a hot swap controller should be used to control the transient currents. Linear Technology makes a family of hot swap controllers including the LTC1643H, LTC1643L and LTC1643L-1. Another approach to hot swapping is to use linear regulators such as the National Semiconductor LP2986 (positive)and the Linear Technology LT1175 (negative). The voltage regulation feedback would be left disconnected or connected to ground so that current limit would be the only feature used. When hot swapped, the maximum peak current drawn on any AutoProbe power bus after the initial 10 useconds shall be <0.6 amps

1. AC means AC coupling. The coupling corner is =/< 10 Hz. It may be implemented using LFR, Low Frequency Reject, or real AC coupling. A TTL low on the Data line will enable AC coupling except when a Digital ID is used.

2. Resistor ID #27 through #40 are currently unassigned IDs. These are reserved for future probes that don't fit into the AutoProbe Standard Probe Look-up Table. This could include current probes, fiber optic probes, probes with different offset voltage ranges, etc. This feature provides an alternative to a Digital ID.

3. In the AutoProbe Standard Look-up Table, the offset voltage range at the input of a 1:1 and a 10:1 probe = +/- 20 volts and a 100:1 probe = +/- 500 volts. To ensure that a calibrated probe has these offset voltage ranges after calibration, the probe shall be designed with 15% excess offset voltage range. i.e. +/-23 volts or +/- 575 volts. Before probe calibration, the PROBE CONTROL AND POWER BOARD assumes this +/- 15% of excess offset voltage range and compensates for it. i.e. it will deliver +/- 870 uAmps of offset current prior to calibration. The oscilloscope via the PROBE CONTROL AND POWER BOARD can determine (when it is calibrating the probe) if another offset voltage range is used by that particular probe and set up the oscilloscope accordingly. The oscilloscope will only be able to distinguish between +/- 5 volt, +/- 10 volt, +/- 20 volt, +/- 100 volt, +/- 200 volt and +/- 500 volt ranges. This would require calibration before the probe is used.

4. The offset voltage polarity shall be such that a positive output offset current (i.e. sourced current) shall cause a negative offset voltage for the probe, and a negative output offset current shall cause a positive offset voltage for the probe. The oscilloscope via the PROBE CONTROL AND POWER BOARD can determine (when it is calibrating the probe) if that particular probe has the opposite polarity offset and can set up the oscilloscope accordingly. This would require calibration before the probe is used.

5. If a Probe ID resistance of >1 meg ohm is measured, the oscilloscope can set up to default settings of 1:1 and 50 ohms. This default was chosen so that the oscilloscope will automatically be set up for coaxial BNC cables that don't have the Probe ID feature.

6. Probe ID resistance tolerances of existing probes is considered to be +/- 10% and all others to be +/- 1%.

7. The default unit of measure is volts for probe resistor ID's 1 through 22. It is as defined for all other probe resistor ID's. e.g. amps, watts, lumens, etc.

3.5.2 Probe ID Resistor ESD protection

Thick film chip resistors are susceptible to ESD damage. An ESD suppressor such as theTR1/0805ESDA from SurgX can be placed in parallel with the Probe ID resistor to provide the protection. Or a capacitor may also be used as long as the R-C time constant is <10 mseconds.

Range: 2 to 100 uamps

3.5.4 Modifying Probe ID Resistor Defaults

The "oscilloscope user unique" default values shown in the Probe ID Resistor Tables can be modified. When a probe is attached to the oscilloscope, an interrupt will occur that will alert the oscilloscope to find out what type of probe is attached and automatically set up the oscilloscope for that probe. Then, if the user desires, he/she can change that setup and store new default values for that value of Probe ID resistor. Then, every time a probe with that value of Probe ID resistor is attached to the oscilloscope, the oscilloscope will set up to the "new" default values.

3.6 INTERRUPTS

There are three types of interrupts that the PROBE CONTROL AND POWER BOARD will send to the oscilloscope through the acquisition board. The oscilloscope will have to poll the PROBE CONTROL AND POWER BOARD to determine what caused the interrupt.

One interrupt will occur as a result of changing probes. The Probe ID Ohmmeter will be left in a state that can detect a Probe ID resistance above or below ~1 meg ohm, for each of the four probe I/F's. When a probe is removed, the PROBE CONTROL AND POWER BOARD will detect an open circuit for the Probe ID resistance (>1 meg ohm) and generate an interrupt for the oscilloscope. The oscilloscope will then set up the default settings for the probe (which are 1:1 and 50 ohms). The oscilloscope will also generate a momentary on-screen message that alerts the user as to what has happened. When a probe is re-attached, another interrupt will be generated because the Probe ID resistance will drop below 1 meg ohm and, once the Probe ID resistance is determined, the oscilloscope will set up for that probe and generate another on-screen message to alert the user. Other messages may be appropriate at this time such as: "Your probe is un-cal'd; you can cal it by connecting it to the cal terminal and . . ."

A second interrupt can come from the Clk line of each of the four probe tips. This interrupt can be a result of a user pushing a button on the probe tip to initiate the interrupt (i.e. AutoAcquire). The oscilloscope will assume that the user wants the oscilloscope to perform a "freeze frame" if the oscilloscope is in the run mode. If it is in the single shot mode, the oscilloscope will perform "trigger arm." This feature will be automatic for probes that do not use Digital ID, whether the probe has the feature or not. The PROBE CONTROL AND POWER BOARD will look for a low going TTL signal (i.e. a switch to ground) on the Clk line to determine when this interrupt occurs.

A third interrupt can come from the Data line of each of the four probe tips on probes that use Digital ID. The details of this mechanism and its application remain undefined at this time.

The maximum voltage that can be measured by the Probe ID Comparators is +5.3 volts. Therefore, the Probe ID Current Source must be less than 100 uamps for resistor IDs > 53k ohms.

3.9 DIGITAL I/O

ERS for the FPGA on the PROBE CONTROL AND POWER BOARD

===================================================================

An Actel 1010B FPGA provides all the digital logic required on the PROBE CONTROL AND POWER BOARD.

All communications with the PROBE CONTROL AND POWER BOARD is done via a 4 wire serial interface; nPrbSerClk, nPrbStb, PrbSerDataIn and PrbSerDataOut. In addition there is a reset signal nReset to the board and an interrupt signal nPrbInt from the board.

The serial port provides an interface to all software programmable features and status on the PROBE CONTROL AND POWER BOARD.

There are 7 4-bit registers and the MULTI-DAC which can be written through the serial interface. In addition there are 7 4-bit registers that can be read through the serial interface.

The acquisition board allows direct clocking of DataIn by writing to an address with D0 used as the serial Data. The Strobe is also pulsed by writing to an address. DataOut must be read in a two part operation. The data is read from status register data bit 14 after separately clocking the serial port.

The acquisition board Clock address offset is AA70

The acquisition board Strobe address offset is 57C0

The acquisition board Status Register address offset is 4410

Probe ID Function

----------------------

The 4 outputs of the Probe ID comparators nPrbID[4:1] (one for each channel) are made available as one of the readable registers on the serial interface. In addition any transition on these lines will generate an interrupt to the system. This function must be enable by setting both data bits 0 and 2 high in the probe control register. Once an interrupt has been detected the Misc register bit 0 can be checked to see if the Probe ID comparators generated an interrupt. Finally the interrupt is cleared by pulsing bit 0 of the probe control register low.

Offset Zero Cal Function

-----------------------------

The 4 outputs of the Offset Zero Calibration comparators ZeroCal[4:1] (one for

each channel) are made available as one of the readable registers on the serial interface.

MULTI-DAC Programming Function

---------------------------------------------

The MULTI-DAC is programmed through the same serial port that accesses the rest of the board. The nDacSerClk, DacData, nDataEn to the MULTI-DAC must first be enabled by setting data bit 1 of the probe control register.

Probe Interface Signals

----------------------------

Each of the four Probe Interfaces will have two control signals named PrbClk and PrbData which can be individually configured as inputs, totem pole outputs, or open collector I/Os. In addition they each can be programmed to generate an interrupt on a falling edge.

Each signal is implemented as a tristateable output buffer with a input on the buffer output node.

The 4 inputs of the PrbClk[4:1] signals (one for each channel) are made available as one of the readable registers on the serial interface. The same is true for the PrbData[4:1].

Each output enable and the input to each PrbClk and PrbData buffer is individually controlled by a register bit. To program the port as an input, the buffer is disabled. To program the signal as a totem pole output the buffer is enabled and the correct data is placed in the buffer. To program the signal as a open collector I/O, the buffer input is low, and the buffer is enabled to generate a 0 and disabled to generate a 1.

Probe Interface Interrupts

-------------------------------

The probe interface signals PrbClk and PrbData can be programmed individually to generate an interrupt on a falling edge. To use a signal as an interrupt line it should be configured as an input, and the appropriate interrupt enable control register bit set. Once an interrupt has been detected the PrbClk or PrbData interrupt status register can be checked to see which interface signal generated the interrupt. Finally the interrupt is cleared by pulsing bit 0 of the probe control register low.

Reset Function

------------------

At reset all interrupts are disabled, the MULTI-DAC interface is off, and all PrbClk and PrbData lines are programmed as inputs.

Probe Interface Applications

----------------------------------

It is pretty much open to what the probe interface signals can be used for, but some examples are:

Control outputs to set active probe coupling modes.

Push button acquisition control on probes.

Digital Probe ID.

Program serial DACs on an active probe.

Store cal factors in a probe.

If these lines are going to be used for a Digital Probe ID, or to implement any other type of serial read it is highly recommended that the two signals be dedicated to running an I^2C serial interface as documented by PHILLIPS. Both lines are used as open collector I/O's and the I^2C protocol is implemented in the controlling software. When implementing the normal I^2C I/O specs the probe should provide 2.15K to +5V pullups on each of the lines.
